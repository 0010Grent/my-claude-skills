# /explain-section 命令

## 命令描述
对论文的指定章节进行深度精读和解析。

## 使用方式
```
/explain-section <章节号> [--depth <basic|deep|expert>]
```

示例：
```
/explain-section 3.4,3.5
/explain-section 3.4 --depth deep
/explain-section 4 --depth basic
```

## 执行逻辑

### Step 1：定位章节
在已加载的论文中定位指定章节内容。

### Step 2：章节解析框架

对每个指定章节，输出以下结构：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📖 章节精读：[章节号] [章节标题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【本章在干什么】
用2-3句话说明本章的核心任务和在全文中的位置。

【核心机制拆解】
将本章的主要机制/方法拆解为步骤：

Step 1：[步骤名称]
 → 做了什么：[描述]
 → 为什么这样做：[原理]
 → 关键细节：[注意点]

Step 2：[步骤名称]
 → 做了什么：[描述]
 → 为什么这样做：[原理]
 → 关键细节：[注意点]

【数学公式解释】
[如果本章有公式，逐一解释]

公式：[原始公式]
白话解释：[用日常语言解释这个公式在做什么]
直觉理解：[为什么要这样设计]
关键参数：[每个参数代表什么]

【算法流程图】（ASCII版本）
[用文字图表示算法流程]

【与其他章节的关联】
- 承接自：[哪个章节的哪个概念]
- 为后续：[为哪个章节铺垫了什么]

【三个关键理解点】
⭐ 理解点1：[读懂这章必须抓住的第一个点]
⭐ 理解点2：[读懂这章必须抓住的第二个点]
⭐ 理解点3：[读懂这章必须抓住的第三个点]

【汇报时的讲解建议】
- 用什么类比来解释这个机制？
- 哪个图表最适合拿来讲？
- 听众最可能在哪里迷惑？如何预防？
```

### Step 3：生成讲解稿草稿

可选：为该章节生成一段口语化的讲解稿，模拟报告人的讲解方式。

## 输出文件
自动保存到 `outputs/[论文标题]/sections/[章节号]_explanation.md`
