# /read-paper 命令

## 命令描述
读取并深度理解一篇 arxiv 论文，输出结构化的论文理解报告。

## 使用方式
```
/read-paper <arxiv链接 或 PDF路径 或 论文文本>
```

## 执行逻辑

当用户调用此命令时，按以下步骤执行：

### Step 1：获取论文
- 如果是 arxiv 链接：调用 `tools/arxiv_fetcher.py` 获取论文内容
- 如果是 PDF 路径：调用 `tools/pdf_parser.py` 解析 PDF
- 如果是粘贴的文本：直接处理

### Step 2：论文结构扫描
提取并输出以下信息：

```
📄 论文基本信息
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
标题：[原文标题]
作者：[作者列表]
机构：[所属机构]
发表时间：[年月]
arxiv ID：[ID号]

🎯 核心贡献（3-5条）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. [贡献1]
2. [贡献2]
3. [贡献3]

📚 论文章节结构
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[自动生成章节目录，每章附一句话说明]

🔑 关键术语表
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[术语英文] (中文解释)：[简短定义]
```

### Step 3：四维深度解析

**维度1：问题定义**
- 论文要解决什么问题？
- 现有方法的局限是什么？
- 为什么这个问题重要？

**维度2：算法核心**
- 核心方法/模型是什么？
- 算法的关键创新点在哪里？
- 训练/推理流程是怎样的？
- 重要公式的直觉解释

**维度3：数据与实验**
- 使用了哪些数据集？
- 实验设计的逻辑是什么？
- Baseline 对比了哪些方法？
- 消融实验验证了什么？

**维度4：结果与意义**
- 主要实验结果是什么？
- 数字背后说明了什么？
- 论文结论的可信度如何？
- 局限性和未来工作是什么？

### Step 4：输出论文地图

生成一个可视化的论文逻辑框架（用文字ASCII图表示）：

```
[问题背景] → [现有局限] → [本文方案]
                               ↓
                   [算法模块1] + [算法模块2]
                               ↓
                   [实验验证] → [结论]
```

### Step 5：推荐精读章节

根据论文内容，推荐哪些章节值得深度精读，并说明理由。

## 输出文件
自动保存到 `outputs/[论文标题]/01_paper_understanding.md`
