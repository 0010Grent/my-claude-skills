# /assign-speakers 命令

## 命令描述
智能分析论文结构，为多位报告人分配合理的讲解板块，确保内容均衡、逻辑连贯。

## 使用方式
```
/assign-speakers [--speakers <数量>] [--total-time <总时长>]
```

示例：
```
/assign-speakers
/assign-speakers --speakers 4 --total-time 60min
/assign-speakers --speakers 2
```

## 执行逻辑

### Step 1：分析论文难度分布

扫描各章节，评估：
- 内容密度（轻/中/重）
- 技术难度（入门/进阶/专家）
- 重要程度（背景/核心/辅助）
- 建议讲解时长

输出难度热力图：
```
章节难度分布
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Section 1: Introduction      ░░░░░  轻量  5分钟
Section 2: Related Work      ░░░░░  轻量  5分钟
Section 3: Method            ████░  较重  20分钟
Section 3.1: Architecture    ███░░  中等  8分钟
Section 3.2: Training        ████░  较重  10分钟
Section 4: Experiments       ███░░  中等  15分钟
Section 5: Analysis          ██░░░  中等  10分钟
Section 6: Conclusion        ░░░░░  轻量  3分钟
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
预计总时长：~58分钟
```

### Step 2：生成分配方案

根据报告人数量，输出最优分配方案（默认3人）：

```
📊 推荐分工方案
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

报告人A — 背景与动机（约15分钟）
  负责章节：1, 2
  主题：论文要解决什么问题，为什么重要
  难度：⭐⭐☆☆☆ 适合负责整体介绍的同学

报告人B — 核心方法（约20分钟）
  负责章节：3, 3.1, 3.2
  主题：算法架构与训练策略
  难度：⭐⭐⭐⭐☆ 建议对该领域有基础的同学

报告人C — 实验与结论（约20分钟）
  负责章节：4, 5, 6
  主题：实验验证与结果分析
  难度：⭐⭐⭐☆☆ 需要理解实验设计逻辑

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
讲解逻辑：A建立问题背景 → B解释解决方案 → C验证效果
```

### Step 3：用户确认与调整

提示用户：
- 是否接受此分工方案？
- 是否需要调整某人的负责章节？
- 报告人是否有特定偏好（如某人更擅长实验分析）？

### Step 4：生成备读清单

确认分工后，为每位报告人生成个人备读清单：

```
📝 报告人B 备读清单
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
必读：
  ✓ Section 3：通读，理解整体架构
  ✓ Section 3.1：精读，掌握模型结构细节
  ✓ Section 3.2：精读，理解训练策略

推荐补读：
  → Section 2.3：了解前置工作，帮助讲解创新点
  → Appendix A：若时间允许，加深对公式推导的理解

重点公式：
  → Eq.(3)：核心损失函数，必须能解释直觉含义
  → Eq.(5)：注意力机制变体，说明与标准版的区别

图表重点：
  → Figure 2：架构图，用于展示整体流程
  → Table 1：消融实验，支撑设计选择的合理性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```
