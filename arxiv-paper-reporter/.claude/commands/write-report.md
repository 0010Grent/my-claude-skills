# /write-report 命令

## 命令描述
基于已理解的论文，生成完整的结构化汇报报告。

## 使用方式
```
/write-report [选项]
```

选项：
```
--speakers <数量>      报告人数量（默认3人）
--duration <时长>      每个板块时长，如 15min、10-20min
--sections <章节>      指定每个报告人负责的章节
--style <style>        报告风格：academic/casual/mixed（默认mixed）
--output <路径>        输出路径
```

示例：
```
/write-report --speakers 3 --duration 15min
/write-report --speakers 2 --sections "A:1,2,3;B:4,5,6"
```

## 执行逻辑

### Step 1：报告整体规划

在生成具体内容前，先输出报告规划供用户确认：

```
📋 汇报报告规划
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

论文：[标题]
总时长：[预计总时长]
报告人数：[X人]

分工方案：
┌─────────────────────────────────────────────────┐
│ 报告人A │ [主题]         │ 章节X.X, X.X │ XX分钟 │
│ 报告人B │ [主题]         │ 章节X.X, X.X │ XX分钟 │
│ 报告人C │ [主题]         │ 章节X.X, X.X │ XX分钟 │
└─────────────────────────────────────────────────┘

逻辑衔接：
A → B → C 的讲解逻辑是：[描述讲解逻辑链条]

是否按此方案生成报告？(确认/调整分工)
```

### Step 2：生成完整报告

用户确认后，生成以下完整报告结构：

---

**报告头部**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
论文解读汇报报告

论文标题：[标题]
论文来源：[arxiv链接]
报告日期：[日期]
报告人：[A、B、C]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**论文概述（5分钟，由报告人A引言）**
```
● 研究背景与动机
● 核心贡献（一句话版本）
● 整体技术路线图
```

**各报告人板块**（使用 CLAUDE.md 中定义的标准格式）

每个板块包含：
- 攻读目标
- 精读章节列表
- 汇报讲解要点（每点含预计时长）
- 听众理解检查点

**报告结尾**
```
🔮 未来展望
● [延伸方向1]
● [延伸方向2]

❓ 推荐讨论问题
1. [问题1]
2. [问题2]
3. [问题3]

📎 参考资源
● 相关论文推荐
● 代码仓库链接（如有）
```

### Step 3：质量检查

生成报告后自动检查：
- [ ] 每个板块时长合理（不超过设定时长）
- [ ] 技术概念均有白话解释
- [ ] 数学公式均有直觉解释
- [ ] 板块间有逻辑衔接语
- [ ] 听众检查点覆盖核心概念

## 输出文件
自动保存到 `outputs/[论文标题]/report_[日期].md`
